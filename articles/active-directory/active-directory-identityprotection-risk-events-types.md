#<a name="types-of-risk-events-detected-by-azure-active-directory-identity-protection"></a>检测到的 Azure 活动目录身份保护的风险事件的类型 

在 Azure 活动目录身份保护，风险事件是事件的︰

- 被标记为可疑

- 表示标识可能被破坏。 

本主题介绍了可用的风险事件类型的详细的概述。


## <a name="leaked-credentials"></a>泄漏的凭据

发布公开在深 web 通过 Microsoft 安全研究人员发现泄漏的凭据。 通常在纯文本中找到这些凭据。 他们也将比照 Azure AD 凭据，如果没有匹配项，它们被报告为"Leaked 凭据"身份保护。

因为他们提供清楚的指示的用户名称和密码的攻击者泄漏风险事件归为"高"严重性风险事件，凭据。

## <a name="impossible-travel-to-atypical-locations"></a>不可能旅行到典型的位置

此风险事件类型标识了两个号接源自遥远的位置，其中至少一个位置也可能是用户的典型，过去的行为给出。 此外，两个符号接之间的时间是较短的时间，则可能需要花费用户从第一个位置移动到第二，表明其他用户正在使用相同的凭据。 

将忽略明显"*误报*"分配给 Vpn 和经常使用的组织中其他用户的位置不可能旅行条件，此机器学习算法。  系统具有初始学习期为 14 天，在此期间了解到新用户的登录行为。

不可能旅行通常是黑客无法成功登录的一个很好指标。 但是，当用户使用新的设备或使用 VPN，通常不由组织中的其他用户在旅行时，可能会出现假阳性。 假阳性的另一个来源是错误地传递作为客户端 Ip 可能为外观的服务器的 Ip 的应用程序正在进行中，该应用程序的后端数据中心承载的登录 （通常这些是的 Microsoft 数据中心，可能会给号接花的外观将从 Microsoft 所拥有的 IP 地址）。 由于这些假阳性，而这风险事件的风险级别是"**中等**"。

##<a name="sign-ins-from-infected-devices"></a>从受感染的设备登录

这种风险事件类型标识登录从恶意软件感染，已知会主动与 bot 服务器进行通信的设备。 这取决于关联的对照与 bot 服务器 IP 地址的用户的设备的 IP 地址。 

此风险事件标识不用户设备的 IP 地址。 如果几种设备位于单个 IP 地址，并且只有一些是受 bot 网络登录的其他设备我触发此事件会不必要地，即分类为"**低**"此风险事件的原因。  

我们建议您联系用户，扫描用户的所有设备。 还有可能，被感染用户的个人设备，或如上文所述，是由他人作为用户使用相同的 IP 地址从感染威胁的设备。 受感染的设备通常是由尚未发现的防病毒软件，也可能表明作为错误的用户习惯可能会导致设备感染病毒的恶意软件感染。

有关如何使用地址恶意软件感染的详细信息，请参阅[恶意软件防护中心](http://go.microsoft.com/fwlink/?linkid=335773&clcid=0x409)。


## <a name="sign-ins-from-anonymous-ip-addresses"></a>从匿名登录的 IP 地址

这种风险事件类型标识用户已成功登录从已被标识为匿名代理服务器 IP 地址的 IP 地址。 使用这些代理服务器时要隐藏其设备的 IP 地址，并可以用于恶意目的的人员。

我们建议您立即联系用户确认他们使用匿名的 IP 地址。 此风险事件类型的风险级别是"**中等**"，因为本身一个匿名的 IP 不是明显表示帐户泄漏。

## <a name="sign-ins-from-ip-addresses-with-suspicious-activity"></a>从可疑活动的 IP 地址登录

这种风险事件类型标识从中很多失败的登录尝试所看到的那样，跨多个用户帐户，在短时间内的 IP 地址。 这与匹配的攻击，使用的 IP 地址的通信模式，明显表示帐户是已经或即将受到破坏。 这是机器学习算法忽略明显"*假阳性*"，例如定期由组织中的其他用户的 IP 地址。  系统有 14 天的初始学习周期，它了解到新用户和新租户的登录行为。

我们建议您联系以验证它们是否实际上已签名被标记为可疑 IP 地址的用户。 此事件类型的风险级别是"**中等**"，因为几个设备可能是背后的相同的 IP 地址，尽管只有一些可能负责可疑的活动。 


## <a name="sign-in-from-unfamiliar-locations"></a>登录不熟悉的位置

此风险事件类型是一种实时登录评估机制，认为过去的签名中的位置 (IP、 纬度 / 经度和 ASN) 来确定新的 / 不熟悉的位置。 系统存储关于用户，通过使用前面的位置信息，并认为这些"熟悉"的位置。 风险甚至尚不熟悉的位置的列表中的位置发生登录时触发。 系统具有初始学习期限为 14 天，在此期间则不会标记为不熟悉的位置的任何新位置。 系统还将忽略登录从熟悉的设备和那些在地理位置上靠近熟悉的位置的位置。 <br>
不熟悉的位置可以提供明显表示攻击者所能试图使用窃取的身份。 当用户在旅行，为了试验新的设备或使用新的 VPN，可能会出现假阳性。 由于这些误报，而此事件类型的风险等级为"**中等**"。
